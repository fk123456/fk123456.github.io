大数据之Hadoop

大数据概念

  1. 指无法在一定时间范围内用常规软件工具进行捕捉、管理和处理的数据集合，是需要新处理模式才能具有更强的决策力、洞察发现力和流程优化能力的海量、高增长率和多样化的信息资产。
  2. 数据存储单位
    1. bit、Byte、KB、MB、GB、TB、PB、EB、ZB、YB、BB、NB、DB
    2. 目前大公司数据量在EB
  3. 主要解决海量数据的采集、存储和分析计算。

大数据特点（4V）

Volume（大量）

  1. 一些大企业数据流接近EB量级

Velocify（高速）

  1. 数据产生的速度非常快，像天猫双十一。

Variety（多样）

  1. 数据分为结构化（数据库/文本为主的数据）和非结构化数据（网络日志、音频、视频、地理信息等）。

Value（低价值密度）

  1. 快速对有价值数据“提纯”成为目前大数据背景下待解决的难题。

大数据应用场景

  1. 抖音推荐你喜欢的视频
  2. 电商广告推荐你喜欢的商品
  3. 人工智能

大数据发展前景

  1. 党十九大提出“推动互联网、大数据、人工智能和实体经济深度融合”。

Hadoop入门

概念

Hadoop是什么

  1. Hadoop是一个由Apache基金会开发的分布式系统基础架构
  2. 主要解决海量数据的存储和海量数据的分析计算问题
  3. Hadoop通常是指一个更广泛的概念–Hadoop生态圈

Hadoop发展历史

  1. Hadoop创始人Doug Cutting，为了实现与Google类似的全文搜索功能。

Hadoop三大发现版本

  1. Apache版本，开源，对入门学习最好
  2. Cloudera版本，内部集成了很多大数据框架，对应产品CDH
  3. Hortonworks版本，文档齐全，对应产品HDP

Hadoop的优势

  1. 高可靠性：Hadoop底层维护多个数据副本，即使某个单元故障，也不会导致数据的丢失。
  2. 高扩展性：在集群间分配任务数据，可方便的扩展数以千计的节点。
  3. 高效性：在MapReduce的思想下，Hadoop是并行工作的，以加快任务处理速度。
  4. 高容错性：能够自动将失败的任务重新分配。

Hadoop的组成

  1. Hadoop1.x
    1. Common辅助工具
    2. HDFS数据存储
    3. MapReduce计算+资源调度
  2. Hadoop2.x
    1. Common辅助工具
    2. HDFS数据存储
    3. Yarn资源调度
    4. MapReduce计算
  3. Hadoop3.x
    1. 在组成上没有变化
  4. HDFS架构概述
    1. 全称Hadoop Distributed File System，是一个分布式文件系统。
    2. NameNode
      1. 负责数据存储情况概述
    3. DataNode
      1. 负责存储具体数据
    4. 2NN
      1. 协助NameNode，每隔一段时间对它元数据备份
  5. YARN架构概述
    1. 全称Yet Another Resource Negotiator，是一个资源协调管理器
    2. ResourceManager：整个集群资源的管理者
    3. NodeManager：单个节点服务器资源管理者
    4. ApplicationMaster：单个任务运行的管理者
    5. Container：容器，相当于一台独立的服务器，里面封装了任务运行所需要的资源，例如内存、CPU、磁盘、网络等。
  6. MapReduce架构概述
    1. Map阶段并行处理输入数据
    2. Reduce阶段对Map结果进行汇总

模板虚拟机准备

  1. 安装epel-release

    1. yum install -y epel-release

  2. 关闭防火墙

    1. systemctl stop firewalld
    2. systemctl disable firewalld.service

  3. 创建foreign用户

    1. useradd foreign
    2. passed xxx

  4. 配置foreign用户具有root权限，方便后期加sudo执行root权限的命令

    1. 在%wheel这行下面添加一行

  5. 创建目录

    1. 切换到foreign用户并且在/opt目录下新建文件夹

克隆

  1. 配置ip地址 vim /etc/sysconfig/network-scripts/ifcfg-ens33

    1. 配置主机名称 vim /etc/hostname
      1. Hadoop102

安装JDK、Hadoop

  1. 传输JDK和Hadoop到服务器上

  2. 安装JDK

    1. 解压 tar -zxvf jdk-8u212-linux-x64.tar.gz -C /opt/module

    2. 配置环境变量

      1. 进入 /etc/profile.d目录下
      2. 创建文件 sudo vim my_env.sh
      1. 激活环境配置 source /etc/profile

  3. 安装Hadoop

    1. 解压 tar -zxvf hadoop-3.1.3.tar.gz -C /opt/module

    2. 配置环境变量

      1. 进入 /etc/profile.d
      2. 进入文件 sudo vim my_env.sh

    3. 激活环境配置 source /etc/profile
